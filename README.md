# mosstat_dashboard
Проект по сбору, обработке и анализу статистики Москвы с Мосстата. Включает парсер данных, очистку, загрузку в PostgreSQL, REST API и дашборд в Cubisio.

Проект автоматизирует сбор, предобработку и анализ статистических данных Москвы с сайта Мосстата. 

## Особенности проекта
1. **Сбор данных:** парсер скачивает Excel/CSV-файлы с сайта Мосстата по ключевым показателям:
   - Индекс потребительских цен (ИПЦ)
   - Доходы, расходы и сбережения населения
   - Уровень бедности
   - Заболеваемость населения
   - Численность медицинских кадров

2. **Предобработка и анализ:** очистка данных, визуализация динамики показателей с помощью Matplotlib и Seaborn, вывод предположений о трендах.

3. **База данных PostgreSQL:** проектирование таблиц и загрузка обработанных данных.

4. **Сервис для работы с данными:** 
   - Автоматическое скачивание и обработка данных
   - Загрузка в базу
   - REST API для получения данных пользователем

5. **Дашборд в Cubisio:** визуализация ключевых показателей (линейные графики, круговые диаграммы, гистограммы) с возможностью анализа динамики и структуры данных.

## Как использовать
🚀 Быстрый старт

1️⃣ Клонирование репозитория

bash
git clone https://github.com/your-username/mosstat_dashboard.git
cd mosstat_dashboard
2️⃣ Установка зависимостей

bash
# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# Установка пакетов
pip install -r requirements.txt
3️⃣ Настройка базы данных

Вариант A: Docker (рекомендуется)

bash
# Запуск PostgreSQL в контейнере
docker-compose up -d postgres

# Проверка
docker-compose ps
Вариант B: Локальный PostgreSQL

bash
# Создание базы данных
createdb mosstat_db

# Обновить .env файл
4️⃣ Настройка окружения

Создайте файл .env в корне проекта:

env
PG_HOST=localhost
PG_PORT=5432
PG_DB=mosstat_db
PG_USER=postgres
PG_PASSWORD=password
VERIFY_SSL=false
5️⃣ Инициализация базы данных

bash
python init_db.py
6️⃣ Запуск парсинга и загрузки данных

bash
# Через Python
python -c "from app.service import refresh_all; refresh_all()"

# Или через API (после запуска сервера)
curl -X POST http://localhost:8000/refresh
7️⃣ Запуск API сервера

bash
uvicorn app.main:app --reload
Сервер будет доступен по адресу: http://localhost:8000

8️⃣ Анализ данных в Jupyter

bash
jupyter notebook notebooks/analysis.ipynb
9️⃣ Подключение Cubisio

Установите Cubisio (по ссылке из письма)
Подключитесь к вашей PostgreSQL базе данных
Создайте дашборды с визуализациями
📡 API Эндпоинты

Метод	Эндпоинт	Описание
GET	/	Информация о сервисе
GET	/health	Проверка здоровья сервиса
POST	/refresh	Запуск обновления всех данных
GET	/cpi/monthly	Помесячные данные ИПЦ
GET	/cpi/period	ИПЦ за период
GET	/cpi/year/{year}	Годовой ИПЦ
GET	/income	Данные о доходах населения
GET	/poverty	Уровень бедности
GET	/morbidity	Данные о заболеваемости
GET	/medical-staff	Численность медицинских кадров
GET	/latest	Последние данные по всем показателям
Примеры запросов

bash
# Получить численность медицинских кадров за 2024 год
curl "http://localhost:8000/medical-staff?start_year=2024&end_year=2024"

# Получить заболеваемость за 2023 год
curl "http://localhost:8000/morbidity?start_year=2023&end_year=2023"

# Получить ИПЦ за период
curl "http://localhost:8000/cpi/period?start_ym=2023-01&end_ym=2023-12"

# Получить последние данные
curl "http://localhost:8000/latest"

🔍 Результаты анализа

Индекс потребительских цен (ИПЦ)

Наблюдается сезонная волатильность с пиками в начале года
В последние годы ИПЦ стабилизировался в диапазоне 100-101%
Доходы населения

Среднедушевые доходы демонстрируют устойчивый рост
Реальные доходы показывают нестабильность, связанную с экономическими кризисами
Уровень бедности

Прослеживается четкая тенденция к снижению доли населения с доходами ниже ПМ
Эффективность социальных программ подтверждается данными
Заболеваемость

Болезни органов дыхания занимают лидирующую позицию (>40% всех случаев)
Рост заболеваемости может быть связан с улучшением диагностики и старением населения
Медицинские кадры

Численность медперсонала растет
Соотношение врачей к среднему медперсоналу требует оптимизации
